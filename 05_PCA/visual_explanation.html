Armando la Matriz W — Paso a Paso

Con datos reales del dataset Wine · Capítulo 5, Raschka

PASO 0 El dataset Wine original

El dataset Wine tiene 178 muestras y 13 características (Alcohol, Malic acid, Ash, etc.). Para que puedas ver cada número, voy a usar las primeras 5 muestras y 3 características: Alcohol, Malic acid y Flavanoids.

X original — datos crudos (sin estandarizar)

Problema: Alcohol va de ~11 a ~14, Malic acid de ~1 a ~5, Flavanoids de ~0.3 a ~5. Las escalas son muy diferentes. Si proyectamos así, Alcohol dominaría solo por tener números más grandes. Por eso el Paso 1 es estandarizar.
# Código equivalente en Raschka (Cap 5):
from sklearn.datasets import load_wine
X, y = load_wine(return_X_y=True)
# X.shape → (178, 13)

PASO 1 Estandarizar los datos

Restamos la media de cada columna y dividimos por su desviación estándar. Así todas las características quedan con media ≈ 0 y varianza ≈ 1. Esto es crucial antes de PCA.

x_std = (x − μ) / σ

Ejemplo con la primera muestra, columna Alcohol:
x_std = (14.23 − 13.00) / 0.81 = 1.52

Estadísticas por columna

X estandarizado (X_std)

¿Ves? Ahora todos los valores fluctúan alrededor de 0 y tienen escalas comparables. Ya ninguna característica domina a las otras por su magnitud.
# Código:
from sklearn.preprocessing import StandardScaler
X_std = StandardScaler().fit_transform(X)

PASO 2 Calcular la matriz de covarianza

La matriz de covarianza mide cómo varían juntas cada par de características. Es una matriz d × d (3×3 en nuestro caso).

La diagonal tiene las varianzas de cada característica (≈1 porque estandarizamos). Los valores fuera de la diagonal son las covarianzas: si son altas positivas, las características se mueven juntas.

Σ = (1/n) · X_stdT · X_std

Dimensiones: (3 × 178) · (178 × 3) = 3 × 3

Matriz de Covarianza Σ (calculada con las 178 muestras)

Lectura rápida: Alcohol y Flavanoids tienen covarianza 0.55 — están bastante correlacionadas positivamente. PCA buscará la dirección que capture esa relación.
# Código:
import numpy as np
cov_mat = np.cov(X_std.T) # shape: (13, 13) con todas las features

PASO 3 Obtener eigenvectors y eigenvalues

Descomponemos la matriz de covarianza en eigenvectors (las direcciones) y eigenvalues (cuánta varianza captura cada dirección).

Σ · v = λ · v

Donde:
  v = eigenvector (dirección de máxima varianza)
  λ = eigenvalue (magnitud de varianza en esa dirección)

Eigenvalues (λ) — ordenados de mayor a menor

Eigenvectors correspondientes — cada columna es un eigenvector

Clave: El primer eigenvector [−0.55, 0.26, −0.79] es la dirección que captura el 55.4% de la varianza total. Los pesos te dicen que esta dirección combina mucho Flavanoids (−0.79) y Alcohol (−0.55), y poco Malic acid (0.26). Es una "super-característica".
# Código:
eigen_vals, eigen_vecs = np.linalg.eig(cov_mat)
# Ordenar de mayor a menor eigenvalue
idx = eigen_vals.argsort()[::-1]
eigen_vals = eigen_vals[idx]
eigen_vecs = eigen_vecs[:, idx]

PASO 4 Construir la matriz de proyección W

Aquí es donde decidimos a cuántas dimensiones reducir. Queremos ir de 3D → 2D, así que tomamos los 2 primeros eigenvectors (los de mayor eigenvalue) y los apilamos como columnas. Eso forma W.

Seleccionamos los k = 2 eigenvectors con mayor λ

W — Matriz de proyección (3 × 2)

Cada columna de W es un eigenvector. W tiene d=3 filas (una por cada feature original) y k=2 columnas (las nuevas dimensiones que queremos).

W = [v₁ | v₂]

Dimensiones: d × k = 3 × 2
Esto es W. No es más que pegar eigenvectors como columnas. El tercer eigenvector (el de menor λ) lo descartamos — solo capturaba el 10.9% de varianza. Con W construida, ya podemos proyectar.
# Código:
k = 2 # queremos 2 componentes
W = eigen_vecs[:, :k] # primeros k eigenvectors como columnas
# W.shape → (3, 2)

PASO 5 Multiplicar X_std · W

Ahora viene la magia. Multiplicamos cada muestra estandarizada por W. Voy a hacer la multiplicación fila por fila para que veas exactamente qué pasa con cada muestra.

X' = X_std · W

(178 × 3) · (3 × 2) = (178 × 2)

¡Pasamos de 3 dimensiones a 2!

Ejemplo detallado: Muestra #1

// Muestra 1 estandarizada:
x₁ = [1.52, −0.56, 1.22]

// Columna 1 de W (eigenvector 1):
w₁ = [−0.55, 0.26, −0.79]

// Producto punto → PC1:
PC1 = (1.52)(−0.55) + (−0.56)(0.26) + (1.22)(−0.79)
PC1 = −0.836 + (−0.146) + (−0.964)
PC1 = −1.946

// Columna 2 de W (eigenvector 2):
w₂ = [0.43, 0.90, 0.09]

// Producto punto → PC2:
PC2 = (1.52)(0.43) + (−0.56)(0.90) + (1.22)(0.09)
PC2 = 0.654 + (−0.504) + (0.110)
PC2 = 0.259

Resultado: [1.52, −0.56, 1.22] → [−1.946, 0.259]
¡De 3 números pasó a 2!

Resultado para las 5 muestras

¿Qué acaba de pasar? Cada muestra con 3 features se convirtió en un punto con solo 2 coordenadas (PC1 y PC2). Cada nueva coordenada es una combinación lineal de las originales, ponderada por los eigenvectors. No eliminamos una columna al azar — extrajimos la esencia.
# Código:
X_pca = X_std.dot(W) # o: X_std @ W
# X_pca.shape → (178, 2)

PASO 6 Resultado: datos en el nuevo espacio

Aquí puedes ver los datos proyectados. A la izquierda: intento de visualizar las 3 dimensiones originales. A la derecha: los 2 componentes principales donde ya puedes ver la separación de clases.

Resumen completo del proceso

X     (178 × 13)   datos originales
  ↓ estandarizar
X_std (178 × 13)   media=0, varianza=1
  ↓ covarianza
Σ     (13 × 13)   cómo se correlacionan las features
  ↓ eigendecomposition
v₁..v₁₃          13 eigenvectors + eigenvalues
  ↓ seleccionar k=2
W     (13 × 2)    ← ESTA ES LA MATRIZ DE PROYECCIÓN
  ↓ multiplicar
X_pca (178 × 2)    datos reducidos, listos para ML
# Todo junto (como en Raschka):
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

X_std = StandardScaler().fit_transform(X)
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_std)

# pca.components_.T ← esto es W internamente
# X_pca.shape → (178, 2)
